## Design Reentry Reference

Use this reference when deciding whether an incompleteness problem belongs in Design or must return to Discovery.

Stay in Design when:

* the problem statement still appears stable
* intended outcomes still appear stable
* major constraints still appear usable
* the ambiguity is about how to solve the problem

Return to Discovery when:

* the problem statement itself may need revision
* intended outcomes are materially unclear or conflicting
* a major constraint is missing, contradictory, or newly discovered
* the design would need to invent requirements
* confidence in a critical Discovery problem area is too weak for the design choice now required

When recommending reentry:

* name what is unstable
* explain why it is a Discovery issue rather than a Design issue
* ask the minimum repair question needed to restore trustworthy framing
