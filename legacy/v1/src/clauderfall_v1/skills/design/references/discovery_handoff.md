## Clauderfall Discovery Handoff Reference

Use this reference when you need the Discovery outcome that Design is inheriting.

Discovery outcome:

* Clauderfall Discovery is considered ready to hand off into Design.
* The problem is framed well enough that Design should not need to invent the product.

What Discovery established:

* Clauderfall is for a single senior engineer.
* The current product boundary is Discovery plus Design.
* Discovery output is organized around problems/themes with cross-cutting sections.
* Discovery carries per-problem confidence using `low` / `medium` / `high`.
* Discovery may include high-level system architecture, but not concrete interfaces or implementation detail.

What remains a Design problem:

* the artifact shape for a design unit
* what structured fields a design unit should carry
* operator workflow around sequencing, overrides, and persistence mechanics where they affect Design behavior
